# 抖音/TikTok 推荐算法技术详解

## 一、推荐系统架构

### 1.1 整体流程

```
召回(Recall) → 粗排(Pre-ranking) → 精排(Ranking) → 重排(Re-ranking) → 展示
```

| 环节 | 作用 | 候选规模 | 典型技术 |
|------|------|----------|----------|
| 召回 | 从海量内容快速筛选候选 | 百万→万级 | 双塔模型、协同过滤 |
| 粗排 | 初步排序，缩小范围 | 万级→千级 | 轻量级模型 |
| 精排 | 精细预测用户行为 | 千级→百级 | Wide&Deep、DNN |
| 重排 | 业务规则调整 | 百级→展示 | 多样性、去重、探索 |

### 1.2 双塔召回模型（Two-Tower Model）

抖音/TikTok最核心的召回技术：

**原理**：
- 用户塔：将用户特征编码为向量
- 内容塔：将视频特征编码为向量
- 通过向量相似度计算进行快速召回

**用户塔输入特征**：
- 历史行为序列（观看、点赞、评论记录）
- 用户画像（年龄、性别、地域）
- 实时行为（当前session行为）
- 社交关系

**内容塔输入特征**：
- 视频内容理解（CV识别的画面内容）
- 文本信息（标题、描述、话题）
- 音频特征（BGM、语音）
- 创作者信息
- 历史互动数据

### 1.3 Wide&Deep 精排模型

**Wide部分（线性模型）**：
- 负责"记忆"能力
- 捕捉特征之间的直接关联
- 处理稀疏特征（如特定用户-特定视频的历史交互）

**Deep部分（深度神经网络）**：
- 负责"泛化"能力
- 学习特征之间的复杂非线性关系
- 发现潜在的用户兴趣

**联合训练**：两部分共同优化，输出用户行为概率预测。

## 二、多目标建模体系

### 2.1 目标类型

| 目标类型 | 具体目标 | 权重特点 |
|----------|----------|----------|
| 短期目标 | 完播、点赞、评论、转发 | 即时反馈，权重较高 |
| 长期目标 | 收藏+复访、关注+追更、搜索+打开 | 预估长期价值 |
| 探索目标 | 新兴趣发现、多样性、长尾内容 | 平衡推荐多样性 |
| 负向目标 | 不感兴趣、举报、退出 | 负权重，需避免 |

### 2.2 行为价值权重设计

不同行为被赋予不同价值权重：

```
完播 > 收藏 > 转发 > 评论 > 点赞 > 观看时长
```

**组合目标示例**：
- 收藏+复访：用户收藏后再次观看，表明内容有长期价值
- 关注+追更：关注后持续观看该创作者，表明建立了稳定关系
- 打开+搜索：通过搜索主动找到内容，表明有明确需求

### 2.3 探索机制

为打破信息茧房，算法设置了探索维度：

1. **多样性打散**：控制相似内容出现频次
2. **多兴趣召回**：同时召回用户多个兴趣领域
3. **长尾兴趣扶持**：给小众内容更多曝光机会
4. **随机探索**：一定比例推荐用户未接触过的类型
5. **社交关系拓展**：基于好友兴趣发现新内容
6. **搜索推荐联动**：结合搜索行为拓展兴趣边界

## 三、流量池详细机制

### 3.1 层级结构

```
┌────────────────────────────────────────────────────────────┐
│                        全站推荐（千万级）                    │
├────────────────────────────────────────────────────────────┤
│                     精品推荐池（~100万）                     │
├────────────────────────────────────────────────────────────┤
│                    高级流量池（30-80万）                     │
├────────────────────────────────────────────────────────────┤
│                    中级流量池（10-15万）                     │
├────────────────────────────────────────────────────────────┤
│                    三级流量池（1-2万）                       │
├────────────────────────────────────────────────────────────┤
│                    二级流量池（3-5千）                       │
├────────────────────────────────────────────────────────────┤
│                    初始流量池（200-500）                     │
└────────────────────────────────────────────────────────────┘
```

### 3.2 晋升机制

**首次分发（冷启动）**：
- 新视频通过审核后进入初始流量池
- 系统分配200-500个活跃用户的曝光机会
- 新账号与大V账号均从零开始竞争

**数据评估**：
- 系统记录完播率、点赞数、评论数、转发数
- 与同层级视频的平均数据比较

**叠加推荐**：
- 数据优于平均水平，进入更高层级流量池
- 每次晋升，推荐量指数级增长
- 持续表现优秀可达到全站推荐

### 3.3 时间衰减

- 视频发布后24-72小时是关键窗口期
- 之后推荐权重逐渐衰减
- 但优质内容可能在较长时间后被"二次推荐"

## 四、标签系统详解

### 4.1 标签类型

**用户标签**：
- 基础属性：年龄、性别、地域、设备
- 兴趣标签：基于历史行为推断的兴趣分类
- 行为标签：活跃时段、互动偏好、观看时长偏好
- 社交标签：社交关系网络特征

**内容标签**：
- 类目标签：视频所属内容类别
- 实体标签：视频中出现的人物、物品、场景
- 情感标签：内容的情感倾向
- 质量标签：制作质量、原创性评估

### 4.2 标签匹配逻辑

```
视频发布 → 系统提取内容标签 → 匹配相似用户标签 → 推送给目标用户
                                    │
                                    ▼
                            评估数据表现
                                    │
                    ┌───────────────┴───────────────┐
                    │                               │
                    ▼                               ▼
              数据表现好                        数据表现差
                    │                               │
                    ▼                               ▼
          减弱标签约束                          停止推荐
          扩大推荐范围
```

### 4.3 标签提取权重

| 来源 | 权重 | 说明 |
|------|------|------|
| 视频画面 | 最高 | AI识别画面内容、物体、场景 |
| 语音/字幕 | 高 | ASR识别语音内容 |
| 标题文案 | 高 | 关键词提取 |
| 话题标签 | 中 | #话题标签 |
| BGM | 中 | 音乐类型、流行度 |
| 账号信息 | 低 | 账号简介、历史内容 |

## 五、TikTok特有机制

### 5.1 社群分组（Cluster）

TikTok独特的用户分组机制：
- 当多个用户对同一视频产生相似行为（如点赞），系统将这些用户归为一个群组
- 之后会向群组成员推荐其他成员喜欢的内容
- 形成基于共同兴趣的内容分发网络

### 5.2 2025年算法更新

| 更新 | 说明 |
|------|------|
| 长视频偏好 | >30秒视频获得更多推荐权重 |
| 质量评分 | AI扫描内容质量、原创性、制作水平 |
| 收藏/分享加权 | 收藏和分享的权重显著提升 |
| 互动质量 | 评论质量比数量更重要 |
| 社群深化 | 微病毒传播（Micro-virality）机制 |

### 5.3 高质量内容优势

TikTok数据显示：高质量视频的粉丝增长速度是低质量视频的40倍。

质量评估维度：
- 画面清晰度
- 音频质量
- 剪辑节奏
- 原创性
- 内容完整性

## 六、算法规避的内容

### 6.1 降权内容类型

- 重复/搬运内容
- 低质量制作（模糊、噪音）
- 误导性/标题党内容
- 过度营销/硬广
- 负面/争议性内容

### 6.2 违规风险

- 版权问题（音乐、画面）
- 敏感话题
- 虚假信息
- 诱导行为
